# CAP-0075 Host Function Wrapping

## Overview

CAP-0075 introduces native BN254 cryptographic host functions into the
Soroban Virtual Machine, enabling gas-efficient ZK proof verification.

## Available Host Functions

| Function | Description | Gas Cost |
|----------|-------------|----------|
| `bn254_g1_add` | G1 point addition | ~500 instructions |
| `bn254_g1_mul` | G1 scalar multiplication | ~50,000 instructions |
| `bn254_multi_pairing_check` | Multi-pairing check | ~1,500,000 instructions |
| `poseidon2_permutation` | Poseidon2 state permutation | ~10,000 instructions |

## Data Translation

### G1 Point (64 bytes)
```
bytes[0..32]  = x coordinate (big-endian Fq element)
bytes[32..64] = y coordinate (big-endian Fq element)
```

### G2 Point (128 bytes)
```
bytes[0..32]   = x.c0 (real part, big-endian)
bytes[32..64]  = x.c1 (imaginary part, big-endian)
bytes[64..96]  = y.c0 (real part, big-endian)
bytes[96..128] = y.c1 (imaginary part, big-endian)
```

### Internal ↔ Host Conversion

```rust
// Internal ethnum::u256 → Soroban BytesN<64>
let mut buf = [0u8; 64];
buf[0..32].copy_from_slice(&point.x.to_be_bytes());
buf[32..64].copy_from_slice(&point.y.to_be_bytes());
let sdk_g1 = Bn254G1Affine::from_bytes(BytesN::from_array(env, &buf));
```

## Error Handling

Host functions panic on invalid inputs (points not on curve, etc.).
The library validates all inputs before calling host functions to convert
host panics into `ZkError` variants.

## Poseidon2 Parameters (BN254, t=3)

```rust
env.crypto_hazmat().poseidon2_permutation(
    &state,          // Vec<U256> of length t=3
    "BN254",         // field identifier
    3,               // t: state width
    5,               // d: S-box degree
    8,               // rounds_f: full rounds
    56,              // rounds_p: partial rounds
    &mat_diag,       // internal matrix diagonal
    &round_constants // 64 round constant rows
)
```
